# GAT 版本迭代计划

> 专业的 Git 可视化管理工具 - 从 0.1.0 到 1.0.0 的完整路线图

## 当前状态评估 (v0.1.0)

### ✅ 已实现功能
- 多仓库扫描与管理
- 文件状态查看（已暂存/未暂存/未跟踪）
- 分支切换、创建、重命名、删除
- AI 辅助提交信息生成（DeepSeek/GLM）
- 分支发布与推送
- 提交历史查看
- Diff 查看
- 批量提交

### ⚠️ 当前不足
- 缺少冲突解决工具
- 无 stash 功能
- 无交互式 rebase
- 代码审查功能薄弱
- 无远程仓库管理（clone/fetch/pull）
- 无标签管理
- 性能优化空间（大型仓库）
- 测试覆盖率为零
- 文档不完善
- 无国际化支持
- 无快捷键系统

---

## 优先级说明

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | Stash、Pull、Clone、冲突解决、测试 | 基础必备功能 |
| P1 | 交互式 Rebase、快捷键、性能优化 | 显著提升体验 |
| P2 | 主题、搜索、Diff 增强 | 锦上添花 |
| P3 | CI 集成、PR 管理、插件系统 | 高级用户需求 |

## 开发建议

### 技术债务清理
1. 重构 Git 操作封装层，统一错误处理
2. 引入日志系统（前后端）
3. 性能监控埋点

### 架构优化
1. 前端组件库抽取（可开源）
2. 后端 Git 抽象层（支持 libgit2 和 git CLI 双实现）
3. 状态管理优化（减少不必要的重渲染）
